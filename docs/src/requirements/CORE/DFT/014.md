# CORE-DFT-014 Lowercase namespaces rejected

## Observation
The implementation enforces uppercase-only namespaces in HRIDs and path parsing, preventing use of lowercase directory names.

## Evidence
- `crates/req-core/src/domain/hrid.rs` `KindString::new` requires all characters be uppercase ASCII, so namespace segments with lowercase letters fail validation.
- Path parsing/canonicalization relies on this uppercase-only enforcement, and `req sync`/`validate` will flag lowercase namespaces as invalid.

## Impact
Repositories using lowercase namespaces cannot parse or sync requirements; canonicalization will attempt to force uppercase, violating USR-012 / SYS-033.

## Expected
Permit lowercase (or mixed-case) namespaces while keeping kinds uppercase, and update parsing, validation, and path handling accordingly.
