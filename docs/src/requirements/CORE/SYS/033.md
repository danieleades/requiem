# CORE-SYS-033 Lowercase Namespace Handling

## Statement
The system shall support lowercase namespaces in HRIDs and filesystem paths, while preserving uppercase enforcement for kinds and zero-padded IDs.

## Scope
- HRID parser/validator accepts namespace segments with lowercase a–z (and mixed case) but rejects lowercase kinds.
- Path parser and path construction functions accept and emit lowercase namespaces without forcing uppercase.
- Canonical path generation maintains namespace casing chosen on disk.
- CLI/MCP layers display namespaces without uppercasing them.

## Verification
- HRIDs with lowercase namespaces (e.g., `auth-api-SYS-001`) parse successfully; kinds remain uppercase.
- Path-based mode constructs and resolves paths using lowercase namespaces.
- `req sync` / `req validate` do not flag lowercase namespaces as invalid.
- Round-trip (parse → display → construct path) preserves namespace casing.
