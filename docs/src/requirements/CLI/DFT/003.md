# CLI-DFT-003 List command header/columns deviate from SPC-002

## Observation
`req list` shows `limit: 0` instead of `limit: unlimited` for `--limit 0`, defaults columns as HRID, Title, Kind, Parents, Children, Tags (spec expects HRID, Kind, Parents, Children, Title), and omits header depth indicators.

## Evidence
- req/src/cli/list.rs:346-356 header formatting, 1071-1077 default columns.

## Impact
The listing UX diverges from SPC-002, reducing consistency and discoverability of filters/views.

## Expected
Render headers/columns/depth metadata per SPC-002, including `limit: unlimited` when the page size is disabled and the prescribed default column order.
