# CLI-DFT-004 Review/suspect CLI missing alias and error semantics

## Observation
`req suspect` alias is absent and `req review --accept` does not emit the SPC-001 usage/exit-64 message when the link is unknown.

## Evidence
- req/src/cli.rs registers only the `review` subcommand; no alias for `suspect`.
- req/src/cli/review.rs:613-655 falls back to generic errors instead of the specified exit 64 guidance for unknown child/parent pairs.

## Impact
Backwards compatibility and scripted flows from SPC-001/SPC-005 are broken and users lack the expected error guidance.

## Expected
Add the `req suspect` alias and enforce the specified exit-64 message when the child/parent pair cannot be found.
