# CLI-SYS-033 Summary Export Command

## Statement

The CLI shall provide an export command (`req export summary`) that regenerates the requirements section of an mdBook `SUMMARY.md` from the requirements graph, writing only within an explicitly marked region of the file.

## Rationale

`SUMMARY.md` typically mixes hand-written chapters with requirement listings, so the tool must not own the whole file. A marker-delimited region (`<!-- requiem:summary:start -->` / `<!-- requiem:summary:end -->`) lets the tool keep requirement navigation synchronized (CORE-SYS-035) while leaving all other content untouched.

## Implementation Notes

- Entries are grouped by namespace and kind using mdBook draft chapters, with one linked entry per requirement (`HRID: Title`)
- Links are relative to the `SUMMARY.md` location and reflect the actual on-disk paths of requirement files
- The target file defaults to `SUMMARY.md` in the requirements root and can be overridden with `--file`
- If the markers are absent, the command fails with guidance rather than guessing an insertion point
- A `--check` flag reports staleness (exit code 2) without writing, for use in CI

## Verification

- Running the command populates the marked region with all requirements, grouped by namespace and kind
- Content outside the markers is byte-for-byte unchanged
- Re-running the command on an up-to-date file makes no changes
- `--check` exits with code 2 when the generated section is stale and 0 when current
- The command errors when markers are missing, duplicated, or malformed
