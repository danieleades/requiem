# MCP-SPC-003 Requirement Kind Metadata Structure

## Statement
Repository configuration shall support optional metadata for each kind that MCP clients can query.

## Fields
1) `description`: explains the kind's purpose.
2) `typical_usage`: guidance on when to create requirements of this kind.
3) `example_hrids`: example identifiers showing the expected format.

## Access
This metadata shall be queryable via the MCP server so agents can understand kind semantics.
