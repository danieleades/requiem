# SYS-028 Validate Command

## Statement

The tool shall provide a unified 'req validate' command that comprehensively checks repository health and integrity.

## Scope

This command consolidates validation previously scattered across status, suspect, and diagnose commands. It performs multiple checks:

1. **Structure check:** Valid YAML frontmatter, required fields, HRID format
2. **Paths check:** Files at canonical locations per path mode
3. **Links check:** No broken references, no circular dependencies
4. **Suspect check:** Parent content hasn't changed (fingerprint drift)

Users can run all checks or specific subsets via --check flag.

## Verification

- Default behavior runs all checks
- --check flag accepts: paths, links, suspect, structure, all
- Multiple --check flags can be combined
- --fix flag attempts automatic repair where safe
- --output flag supports: table, json, summary
- --quiet flag suppresses output (CI usage)
- Exit codes: 0 (no issues), 2 (issues found without --fix)
- Reports list all found issues with actionable guidance
- JSON output includes structured issue data for tooling
